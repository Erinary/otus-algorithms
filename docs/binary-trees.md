### Производительность двоичного дерева поиска

| Заполнение дерева (N = 25000): | Random BST | Ascending BST | 
|--------------------------------|------------|---------------|
| создание                       | 34 ms      | 3442 ms       |
| поиск                          | 4 ms       | 135 ms        |
| удаление                       | 1 ms       | 216 ms        |

Большое время создания Ascending BST связано с тем, что при вставке элементов в порядке возрастания это дерево по сути 
выраждается в linked list (и вставка займет N), в то время как у "нормального" дерева вставка занимает log (N). Плюс 
дополнительная проблема с вырожденным деревом - можно получить переполнение стэка (поэтому выбрано такое значение N, на 
больших - уже ошибка StackOverflow).
Аналогично с поиском и удалением - где у нормально распределенного дерева операция занимает log (N), у вырожденного - N.
